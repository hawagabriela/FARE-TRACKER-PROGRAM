# DETAILED METHODOLOGY SECTION
-- This section is a documentation file of clear, full description of our program, so the user can have a proper understanding and she/he will be able to know how to use the Fare Tracker Program.

## IMPLEMENTATION OF CORE FEATURES ( how are the features applied ) 
 The features are applied using basic Python logic. The program collects user input and processes it step by step;
- Users input fare details ( date, transport, amount) and allocate for the weekly budget

- Data is stored in the list

- Loops are used to display all fares and compute total

- The program checks if weekly budget is exceeded

- A receipt will be shown containing user's fare details

## TECHNOLOGIES INVOLVED ( the set of tools applied in the system )
The system is built using Python programming language. It uses basic data structures like lists to store information. It also uses loops, conditions, and input/output functions to make the program interactive. The program runs in a terminal or Python IDE.

## BACKEND-FRONTEND COMMUNICATION ( how the system’s interface and processing components exchange data and interact )
The program uses a simple text-based interface;
- Frontend: shows a menu where users choose options and input necessary data

- Backend: processes data and performs calculations

- Data will be stored in a list

- Results will be displayed back in the terminal

## KEY DESIGN DECISION AND TRADE-OFF ( why this design is chosen )
We chose a simple list-based design to make the program easy to build and understand, making the system beginner-friendly and fast to use. However, one limitation is that the data is not saved permanently after the program is closed. We accepted this trade-off in order to keep the program simple and focused on basic functionality.

## REFERENCE TO ETHICAL CONSIDERATIONS IN PROGRAMMING CHOICES  (e.g., user privacy, accessibility)
Our program considers basic ethical principles in its design. It does not collect or store any personal or sensitive information, only the fare data entered by the user. The interface is also simple and accessible, making it easy for different users to understand and use. We designed the program to be clear, safe, and user-friendly.
